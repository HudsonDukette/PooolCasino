Implement a Player-Owned Casino Hub system that integrates with the existing multiplayer casino app and introduces a controlled economy with money sinks.

🏗 CORE CONCEPT
Each casino is a hub/environment, not just a game list
Casinos have their own:
Games tab
Economy (bankroll)
Player activity
Players enter casinos and interact within them like mini-platforms
💰 CASINO CREATION (ECONOMY SINK)
Creating a casino costs: 100,000,000 (100M)
When purchased:
The full amount is sent back to the main pool (money sink)
Limit:
One casino per player (configurable)
🎮 GAME OWNERSHIP SYSTEM
Casino owners do NOT automatically get all games
Each game must be purchased individually
Cost per game: 1,000,000 (1M)

When purchased:

Money is sent back to the main pool
Game becomes available in that casino’s “Games” tab
🧱 CASINO HUB STRUCTURE

Each casino has:

Header (like main app top bar)
Tabs:
Games
Stats
Logs
Owner Controls
Unique identity (name, theme, owner)
💵 CASINO BANKROLL SYSTEM
Casino owners must fund their casino manually
Bankroll is used to:
Pay out player wins
If bankroll is too low:
Bets are restricted automatically
📉 MONTHLY TAX (CRITICAL ECONOMY CONTROL)
Every month:
10% of each casino’s current bankroll is removed
Sent back to the main pool
This must run automatically via a scheduled job

Casino owners can:

Set:
Minimum bet
Maximum bet

Pause / unpause casino

Deposit / withdraw bankroll
